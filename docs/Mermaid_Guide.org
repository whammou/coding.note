#+name: Description
#+begin_quote org
Mermaid's syntax is used to create diagrams. =syntax=, together with =deployment= and =configuration= constitute the whole of mermaid
#+end_quote

Diagrams examples can be found in the [[https://mermaid.live][Mermaid Live Editor]]

*** Flowcharts Diagrams :subMermaid@Diagram@Flowchart:
:PROPERTIES:
:ID: 3450a0b4-944d-4ea3-929c-09464cdbd232
:CUSTOM_ID: flowcharts-diagrams

Flowcharts are composed of =nodes= (geometric shapes) and =edges= (arrow and lines). The mermaid code defines how nodes and edges are made and accommodates different arrow types

#+name: Warning Admonition - breakcase
#+begin_quote org
[!WARNING]
Using the word "end" (in all lowercase) in a Flowchart node will break the flowchart. To workaround use the entire word of any of the letters (e.g., "End" or "END")
#+end_quote

#+name: Warning Admonition - letter breakcase
#+begin_quote org
[!WARNING]
If using the letter "o" or "x" as the first letter in a connecting flowchart, add a space before the letter or capitalize the letter (e.g.g, "dev--- ops", "dev---Ops").\\
Typing "A---oB" will create a circle node.\\
Typing "A---xB" will create a cross node
#+end_quote

#+name: Breaking wards and symbols
#+begin_quote org
[!WARNING]
One should beware that the use of some words or symbols that can break diagrams. These words or symbols are few and often only effect specific types of diagram.
#+end_quote

| Diagram Breakers   | Reason                                                             | Solution                                         |
|--------------------+--------------------------------------------------------------------+--------------------------------------------------|
| comments           | N A                                                                | N A                                              |
| %%(``)%%           | Similar to Directive confuses the renderer                         | In comments using %%, avoid using "{}"           |
| flow-charts        | N A                                                                | N A                                              |
| 'end'              | The word "End" can cause flowcharts and Sequence diagrams to break | wrap them in quotation marks to prevent breakage |
| Nodes inside Nodes | Mermaid gets confused with nested shapes                           | wrap them quotation marks to prevent breaking    |
